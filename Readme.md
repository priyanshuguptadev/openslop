# Openslop Monorepo

Welcome to **Openslop** – a modern TypeScript monorepo powered by **Turbo** and **Bun**. The repository is organized using Yarn / npm workspaces and contains a set of reusable packages and example applications.

## 📂 Repository Structure

```
.
├─ apps/                # Example applications (CLI, landing page, …)
├─ packages/            # Re‑usable packages
│   ├─ agent/           # Core agent logic
│   ├─ config/          # Shared configuration (eslint, prettier, …)
│   └─ typescript-config/ # Base TS config shared across the repo
├─ turbo.json           # Turborepo pipeline definition
├─ package.json         # Root package (dev dependencies, workspaces)
└─ Readme.md            # This file
```

### Packages

- **`@openslop/agent`** – The main library that powers the Openslop AI‑assistant.
- **`@openslop/config`** – Centralised ESLint, Prettier, and other tool configurations.
- **`@openslop/typescript-config`** – Base `tsconfig.json` that all packages extend.

### Apps

- **`apps/cli`** – A command‑line interface built with TypeScript and `tsup`.
- **`apps/landing`** – A Next.js landing page showcasing the project.

## 🚀 Getting Started

1. **Install dependencies**

   ```bash
   bun install   # or `npm install` / `yarn`
   ```

2. **Build all packages**

   ```bash
   bun run build   # runs `turbo run build`
   ```

3. **Run an example app**

   ```bash
   # CLI example
   bun run start:cli   # defined in apps/cli/package.json

   # Next.js landing page
   cd apps/landing
   bun dev
   ```

## 🛠 Development

- **Watch mode** – Turbo will automatically rebuild packages that change.
  ```bash
  bun run dev   # equivalent to `turbo run dev`
  ```
- **Lint & format**
  ```bash
  bun lint          # runs ESLint across the repo
  bun format        # runs Prettier
  ```
- **Testing** – Add your tests under `*/src/**/*.test.ts` and run:
  ```bash
  bun test
  ```

## 📦 Publishing

Packages are versioned independently using conventional commits. When ready, run:

```bash
bun publish
```

This will publish all changed packages to the npm registry.
